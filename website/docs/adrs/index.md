# Architecture Decision Records

Architecture Decision Records (ADRs) capture the key architectural decisions made during the development of Claude Ops. Each ADR documents the context, decision drivers, considered options, and the chosen approach with its trade-offs.

| ADR | Decision |
|-----|----------|
| [ADR-0001](adr-0001) | Tiered model escalation (Haiku → Sonnet → Opus) |
| [ADR-0002](adr-0002) | Markdown as executable instructions |
| [ADR-0003](adr-0003) | Prompt-based permission enforcement |
| [ADR-0004](adr-0004) | Apprise notification abstraction |
| [ADR-0005](adr-0005) | Mounted repo extension model |
| [ADR-0006](adr-0006) | MCP infrastructure bridge |
| [ADR-0007](adr-0007) | JSON file cooldown state |
| [ADR-0008](adr-0008) | Go + HTMX web dashboard |
| [ADR-0009](adr-0009) | Docker Compose deployment |
| [ADR-0010](adr-0010) | Claude Code CLI subprocess |
| [ADR-0011](adr-0011) | Session page CLI output and response |
| [ADR-0012](adr-0012) | Browser automation for authenticated web UIs |
